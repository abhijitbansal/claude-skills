# WS1 — Release v2 (Fastlane + Xcode Cloud) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the ios-dev release skill on app.yml v2 + Fastlane so `/release testflight|appstore` works on any configured app, with the three mining-driven pre-flight gates (runtime-trap audit, compliance asserts, entitlement parity) that would have stopped the crashes that shipped.

**Architecture:** SKILL.md orchestrates 9 stages; deterministic checks live in `scripts/` (each independently testable via bats with fixture repos); irreversible upload steps go through Fastlane (`bundle exec fastlane`). Per-app divergence = app.yml values + optional `scripts/release-hooks/<stage>-{pre,post}.sh` in the app repo.

**Tech Stack:** bash 3.2-compatible scripts, Fastlane (gym/pilot/deliver) invoked but NOT vendored (templates land in WS2 scaffold), bats.

## Global Constraints

- Spec §6. Depends on WS0 (`load_app_config.sh` v2 exports: `RELEASE_*`, `TARGETS_*`, `SITE_*`) and references WS5 P0 skills by name in gate output.
- Never hand-edit MARKETING_VERSION/CURRENT_PROJECT_VERSION outside project.yml; never agvtool (rule is executable in bump script).
- Gates WARN by default; hard-FAIL only on: dirty tree, compliance asserts (usage strings/encryption flag missing), entitlement parity mismatch, whatsnew missing entry (appstore mode). Runtime-trap grep is WARN-only (false positives expected; per-app suppress via `release.hooks_dir` pre-hook exiting 0 after review — no extra suppress-list config, YAGNI).
- Every script sources `load_app_config.sh` and starts with `validate_app_config.sh` (stage 0).
- Keep the existing `release.sh` upload-confirmation UX (explicit typed confirmation before upload) — port it, don't regress it.
- bats fixtures: build a minimal fake app repo in `$TMP` per test (project.yml + generated-plist stub + entitlements stubs) — see Task 1 Step 1 for the canonical fixture; reuse via a `make_fixture_app` helper in `tests/bats/helpers.bash`.

- [ ] **Step 3: Implement** `preflight.sh` (gate framework + the gates above):

```bash
#!/usr/bin/env bash
# Release pre-flight: deterministic gates before any build/upload.
# Output contract: PASS|WARN|FAIL: <gate>[: detail]. Exit 1 iff any FAIL.
set -uo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
# shellcheck source=../../_lib/load_app_config.sh
source "${SCRIPT_DIR}/../../_lib/load_app_config.sh"
bash "${SCRIPT_DIR}/../../_lib/validate_app_config.sh" >/dev/null || {
  echo "FAIL: app-config: validate_app_config failed"; exit 1; }

MODE="testflight"
[[ "${1:-}" == "--mode" ]] && MODE="${2:-testflight}"

fails=0
pass() { echo "PASS: $1"; }
warn() { echo "WARN: $1: $2"; }
fail() { echo "FAIL: $1: ${2:-}"; fails=$((fails + 1)); }

# --- tree-clean
if [[ -z "$(git status --porcelain)" ]]; then pass tree-clean
else fail tree-clean "uncommitted changes"; fi

# --- branch (warn off-main; releases may cut from main by convention)
branch="$(git rev-parse --abbrev-ref HEAD)"
[[ "${branch}" == "main" ]] && pass branch || warn branch "releasing from '${branch}', not main"

# --- toolchain gates (skippable in tests)
if [[ "${PREFLIGHT_SKIP_TOOLCHAIN:-}" != "1" ]]; then
  if command -v xcodegen >/dev/null && [[ -f project.yml ]]; then
    xcodegen generate --quiet && pass xcodegen-fresh || fail xcodegen-fresh "xcodegen generate failed"
  else
    warn xcodegen-fresh "xcodegen or project.yml missing"
  fi
  if security find-identity -v -p codesigning | grep -q "Apple Distribution"; then
    pass signing-identity
  else
    fail signing-identity "no 'Apple Distribution' identity in keychain"
  fi
fi

# --- fonts (only when configured)
if [[ "${RELEASE_FONTS_EXPECTED}" -gt 0 ]] 2>/dev/null; then
  count="$(find . -name '*.ttf' -not -path './build/*' | wc -l | tr -d ' ')"
  [[ "${count}" -eq "${RELEASE_FONTS_EXPECTED}" ]] \
    && pass fonts \
    || fail fonts "expected ${RELEASE_FONTS_EXPECTED} .ttf, found ${count}"
else
  pass fonts
fi

# --- generated-plist compliance gates
PLIST="${PREFLIGHT_PLIST:-}"
if [[ -z "${PLIST}" ]]; then
  PLIST="$(find build -name "${APP_NAME}-Info.plist" 2>/dev/null | head -1)"
fi
if [[ -n "${PLIST}" && -f "${PLIST}" ]]; then
  for key in ${RELEASE_USAGE_STRINGS}; do
    grep -q "<key>${key}</key>" "${PLIST}" \
      && pass "usage-strings" \
      || fail "usage-strings" "${key} missing from generated plist"
  done
  [[ -z "${RELEASE_USAGE_STRINGS}" ]] && pass usage-strings
  if grep -q "ITSAppUsesNonExemptEncryption" "${PLIST}"; then
    pass encryption-flag
  else
    fail encryption-flag "ITSAppUsesNonExemptEncryption not declared (app.yml release.encryption_exempt=${RELEASE_ENCRYPTION_EXEMPT})"
  fi
  if [[ -n "${RELEASE_REQUIRED_CAPABILITIES}" ]]; then
    for cap in $(sed -n 's/.*UIRequiredDeviceCapabilities.*//p' "${PLIST}"); do :; done
    # allow-list check: any capability present in plist must be in the allow-list
    caps_in_plist="$(python3 -c "
import plistlib,sys
d=plistlib.load(open('${PLIST}','rb'))
print(' '.join(d.get('UIRequiredDeviceCapabilities',[])))" 2>/dev/null || true)"
    bad=""
    for c in ${caps_in_plist}; do
      case " ${RELEASE_REQUIRED_CAPABILITIES} " in *" ${c} "*) ;; *) bad="${bad} ${c}";; esac
    done
    [[ -z "${bad}" ]] && pass capabilities || fail capabilities "unexpected UIRequiredDeviceCapabilities:${bad}"
  else
    pass capabilities
  fi
else
  warn usage-strings "no generated plist found — build once, or set PREFLIGHT_PLIST"
fi

# --- entitlement parity (App Group identical across app + extensions)
if [[ -n "${TARGETS_APP_GROUP}" ]]; then
  ENT_DIR="${PREFLIGHT_ENTITLEMENTS_DIR:-build/gen}"
  for target in "${APP_NAME}" ${TARGETS_EXTENSIONS}; do
    f="${ENT_DIR}/${target}.entitlements.groups"
    if [[ -f "${f}" ]]; then
      grep -qx "${TARGETS_APP_GROUP}" "${f}" \
        && pass entitlement-parity \
        || fail entitlement-parity "${target}: app group != ${TARGETS_APP_GROUP}"
    else
      warn entitlement-parity "${target}: no extracted entitlements at ${f}"
    fi
  done
else
  pass entitlement-parity
fi

[[ ${fails} -gt 0 ]] && { echo "preflight: ${fails} failure(s)"; exit 1; }
echo "preflight: all gates green"
exit 0
```

HEAVY_PAT='backfill\(|pdfData\(|Data\(contentsOf|VNImageRequestHandler|MLModel|jpegData\(|buildFloorPlan|embed'
hits=""
for f in $(grep -rlE 'App\b.*:.*App|@main' --include='*.swift' Sources 2>/dev/null); do
  # lines inside init/.task/onAppear blocks that call heavy patterns and are not
  # inside Task.detached / TaskGroup / nonisolated context on the same line-window

    NR<=inblock && $0 ~ pat && $0 !~ /Task.detached|withTaskGroup|nonisolated/ {
      printf "%s:%d\n", FILENAME, NR
    }' "${f}"

if [[ -s /tmp/rt_hits.$$ ]]; then
  while IFS= read -r hit; do
    warn runtime-trap "possible main-actor heavy work at ${hit} — see skill mainactor-launch-watchdog-audit"
  done < /tmp/rt_hits.$$
else
  pass runtime-trap
fi

```bash
#!/usr/bin/env bash
# Bump MARKETING_VERSION / CURRENT_PROJECT_VERSION in project.yml — the only
# place versions live (never agvtool, never the pbxproj).
set -euo pipefail

KIND="${1:?usage: bump_version.sh {major|minor|patch|build}}"
[[ -f project.yml ]] || { echo "no project.yml here — version bumps happen in project.yml only" >&2; exit 1; }

mv_line="$(grep -E '^\s*MARKETING_VERSION:' project.yml | head -1)"
b_line="$(grep -E '^\s*CURRENT_PROJECT_VERSION:' project.yml | head -1)"
[[ -n "${mv_line}" && -n "${b_line}" ]] || { echo "MARKETING_VERSION/CURRENT_PROJECT_VERSION not found in project.yml" >&2; exit 1; }

mv="$(echo "${mv_line}" | sed -E 's/.*MARKETING_VERSION:[[:space:]]*"?([0-9.]+)"?.*/\1/')"
build="$(echo "${b_line}" | sed -E 's/.*CURRENT_PROJECT_VERSION:[[:space:]]*"?([0-9]+)"?.*/\1/')"
IFS=. read -r maj min pat <<EOF
${mv}
EOF
pat="${pat:-0}"

case "${KIND}" in
  major) maj=$((maj + 1)); min=0; pat=0 ;;
  minor) min=$((min + 1)); pat=0 ;;
  patch) pat=$((pat + 1)) ;;
  build) ;;
  *) echo "unknown bump kind: ${KIND}" >&2; exit 2 ;;
esac
new_mv="${maj}.${min}.${pat}"
new_build=$((build + 1))

sed -i.bak -E "s/(MARKETING_VERSION:[[:space:]]*\"?)[0-9.]+(\"?)/\1${new_mv}\2/" project.yml
sed -i.bak -E "s/(CURRENT_PROJECT_VERSION:[[:space:]]*\"?)[0-9]+(\"?)/\1${new_build}\2/" project.yml
rm -f project.yml.bak
echo "version=${new_mv} build=${new_build}"
```
